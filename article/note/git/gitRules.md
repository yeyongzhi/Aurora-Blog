# Git基础知识
## Git 提交文案规范
提交文案格式：`type: 提交文案`

`type`的种类及其说明

- feat / feature：新功能开发、迭代
- fix：修复程序中的错误或异常行为
- docs：文档更新
- style：代码格式调整（不影响逻辑，如空格、分号）
- refactor：重构
- perf：性能优化（提升代码执行效率）
- test：添加、修改测试
- build：构建系统或依赖更新（如 webpack、npm）
- ci：CI配置或脚本变更
- chore：杂项（如更新 .gitignore）
- revert：回滚之前的提交
